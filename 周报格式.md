# 总体要求
1. **描述信息**：类似SKILL.md文件，包括填报人姓名、填报周期
2. **工作分类**：包括金融研发，寄递研发，管理赋能，数据分析，信息资源管理，队伍建设及业绩融合，下周计划等板块
3. **填写规范**：每个分类的填写标准都是按照项目名称，开始时间，结束时间和工作内容进行编写
4. **时间要求**：开始时间和结束时间按照“2026年3月15日”这样的格式填写，如果在填报日期该项内容仍未完成，结束时间为空

## 项目名称规则
> 通过标准化的编写和检查手段skills来实现
1. **标准名称**：金融研发，寄递研发，管理赋能和数据分析按照项目名称进行填写，其他板块按照标准名称填写
2. **统一规则**：周报中涉及的项目名称，项目组内不同人员填写时要保持一致，每周填写的时候保持统一
3. **业务部门**：周报中涉及的机构和部门，按照标准名称进行描述，例如：省寄递业务部，不要写成省寄递部，版纳州分公司，不要写成版纳公司

## 工作分类
1. **金融研发**：涉及金融业务的相关项目研发，业务需求部门以金融部门为主
2. **寄递研发**：涉及寄递业务的相关研发，业务需求部门包括寄递业务部、运管部、速递物流和运管部等
3. **管理赋能**：非金融和寄递外的其他研发项目，例如：职工小家管理、工程监控、文创产品管理和融资E等，也包括一些软件工具的开发：例如：工具软件的开发：人资的薪酬发放、热力图等等
4. **数据分析**：专项的数据分析项目，例如：抖音指数提升，省内时限对标等
5. **信息资源管理**：涉及省内云平台、超融合平台、集团云平台、数据中台和数据下载平台的运行运维、数据提取和系统迁移等工作
6. **队伍建设和业绩融合**：涉及需求对接、方案研讨、技术交流、技术支持、材料撰写、软件著作权、数据上报和培训等工作，培训只填写需要在工作时间开展的培训，例如：出差的培训、需要到培训中心开展的培训等
7. **下周工作**：下周工作不分板块，按照项目名称进行填写

## 周报文件名要求：

人名 + 工作周报 + 填写日期.md
示例：郭思文工作周报20260326.md
