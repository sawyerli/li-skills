# validate-report

校验一份或多份个人工作周报文件，检查格式、结构和内容是否符合规范，并输出问题清单。

## 格式规范来源

校验依据：当前 skill 目录下的 `references/个人周报格式.md`。执行前必须先读取该文件，以其中的规则为准。

## 触发方式

用户提供待校验的周报文件路径（一个或多个），例如：

```
/validate-report test/郭思文工作周报20260326.md
/validate-report test/
```

如果用户提供的是目录，则对目录下所有 `.md` 文件逐一校验。
但进入校验前，必须先按文件名模式 `人名工作周报YYYYMMDD.md` 过滤；不匹配该模式的文件直接跳过，不纳入校验范围。

## 执行步骤

### Step 1：收集待校验文件

- 如果输入是单个文件，先检查文件名是否匹配 `人名工作周报YYYYMMDD.md`；匹配后再进入校验，不匹配则提示“文件名不符合周报命名模式，未纳入校验”。
- 如果输入是目录，先扫描目录下文件名匹配 `人名工作周报YYYYMMDD.md` 的 `.md` 文件，只对匹配文件逐一校验。
- 目录中的说明文档或其他不符合命名模式的文件，统一跳过。
- 如果过滤后没有待校验文件，直接输出“未找到符合命名模式的周报文件”。

### Step 2：读取规范

读取 `references/个人周报格式.md`，确认以下规则后再开始校验：
- 文件名格式
- YAML front matter 字段和日期格式
- 板块顺序和板块名称
- 表格表头字段（含下周计划的特殊表头）
- 空板块写法
- 日期格式要求
- 当前状态建议值

### Step 3：逐文件校验

对每份周报，依次检查以下各项：

#### 2.1 文件名
- 是否符合 `人名工作周报YYYYMMDD.md` 格式
- YYYYMMDD 是否为有效日期
- YYYYMMDD 是否与 front matter 中 `填报周期结束` 一致

#### 2.2 YAML front matter
- 是否存在且位于文件开头
- 是否包含 `姓名`、`填报周期开始`、`填报周期结束` 三个字段
- 日期是否为 `YYYY-MM-DD` 格式
- `填报周期结束` 是否晚于或等于 `填报周期开始`

#### 2.3 板块完整性与顺序
- 是否包含全部 7 个板块：金融研发、寄递研发、管理赋能、数据分析、信息资源管理、队伍建设及业绩融合、下周计划
- 板块顺序是否正确
- 板块标题是否为二级标题 `##`

#### 2.4 空板块写法
- 无内容的板块是否在标题下写了单独一行 `无`
- 无内容的板块是否错误地写了空表格或直接空白
- `下周计划` 是否写了 `无`（不允许）

#### 2.5 表格格式
- 有内容的板块是否使用了标准 Markdown 表格
- 非下周计划板块表头是否完整：`项目名称 | 开始时间 | 结束时间 | 当前状态 | 工作内容 | 工作成果`
- `下周计划` 表头是否正确：`项目名称 | 开始时间 | 结束时间 | 当前状态 | 工作内容`（不含工作成果）
- 分隔行是否为标准格式 `| --- | --- | ... |`

#### 2.6 日期字段
- 每行 `开始时间` 是否为 `YYYY-MM-DD` 格式
- 每行 `结束时间` 是否为 `YYYY-MM-DD` 格式或空（进行中可留空）
- `下周计划` 每行的 `开始时间` 是否晚于 `填报周期结束`

#### 2.7 当前状态
- 建议值范围为：`已完成`、`进行中`、`已提交待确认`、`已上线`、`已提测`、`待启动`
- 如果使用了非建议值，输出警告，不计入错误数量

#### 2.8 内容规范
- `工作内容` 是否含有模糊表述，如：`做了一些优化`、`处理相关问题`、`持续推进若干工作`
- `工作成果` 与 `工作内容` 是否明显重复
- 一行中是否混入多个项目名称或多个部门名称

### Step 4：输出问题清单

每份文件的校验结果按以下格式输出：

```
## 文件：郭思文工作周报20260326.md

通过项目：文件名、front matter、板块完整性、表格格式、日期格式

问题列表：
- [2.6 日期字段] 下周计划 > 材料编写：开始时间 `2026-03-24` 未晚于填报周期结束日期 `2026-03-26`

警告列表：
- [2.7 当前状态] 寄递研发 > 车牌寄递辅助系统：当前状态 `在途` 不在建议值范围内，建议改为标准表达

结论：存在 1 个问题，需要修改后重新提交。
```

如果文件全部通过，输出：

```
## 文件：郭思文工作周报20260326.md

全部通过，格式合规。

警告列表：
- 无
```

### Step 5：汇总

所有文件校验完成后，输出一行汇总：

```
校验完成：共 N 份文件，M 份合规，K 份存在问题，W 份存在警告。
```

## 注意事项

- 校验时以规范文件为准，不要凭记忆判断规则。
- 日期合法性校验需考虑月份天数（如 2 月不超过 28/29 天）。
- 模糊表述检测不要过度误报，仅标记明显的无实质内容描述。
- 问题描述需定位到具体板块和行，方便填报人修改。
- 排序规则留待后续“周报合并” skill 处理，本 skill 不检查排序。
- 目录输入场景下，只有文件名匹配 `人名工作周报YYYYMMDD.md` 的文件才纳入校验统计。
